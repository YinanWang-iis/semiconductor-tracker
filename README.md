# 半导体与存储产业链追踪网页

这是一个可直接打开的静态网页原型，用来帮助家人按产业链节点追踪半导体和存储行业变化。

## 怎么打开

双击 `index.html`，或在浏览器里打开：

```text
G:\semiconductor_tracker\index.html
```

## 当前功能

- 按产业链拆分：上游材料与设备、芯片设计、晶圆制造、存储制造、先进封装、光模块与服务器、下游需求。
- 按产品节点追踪：AI GPU、HBM、CoWoS、光模块、NAND、DRAM、先进制程、材料、设备等。
- 每个节点统一看三类变化：定价、库存、产能，并展示区间值、单位、QoQ/YoY变化和数据口径。
- 支持按地区、环节、指标和关键词筛选。
- 支持“摘要模式”，减少干扰信息。
- 支持导出当前筛选结果为 CSV。
- GitHub Actions 每天自动生成 `data/latest.json`，网页会优先读取这份自动数据。
- 每日简报：自动给出今天优先看的节点和主要驱动。
- 变化提醒：用红黄绿标记价格、库存、产能或新闻密度变化。
- 来源可信度：把自动抓取证据区分为公司官方、行业机构、专业媒体、一般媒体。
- 历史趋势图：每个节点展示价格、库存、产能的90天趋势小图。
- 为什么变化：按节点展示主要驱动因素，例如AI服务器需求、原厂控产、先进封装瓶颈。
- 影响链条：展示一个节点可能传导影响到哪些上下游节点。
- 自定义关注清单：可把重点节点加入关注，首页简报优先围绕关注节点生成。
- 数据源管理：显示每个节点的自动搜索关键词、证据数量和来源类型。
- 相关公司股市信息：按美股、日股、A股、台股分组展示代表公司的股价、涨跌幅、交易所、对应产业链节点和近60个交易日走势。

## 自动更新机制

自动更新分三层：

1. `data/seed.json` 保存产业链节点、基准量化口径和搜索关键词。
2. `scripts/update-data.mjs` 每天抓取公开新闻索引，把新闻证据归入对应节点。
3. `.github/workflows/update-data.yml` 每天自动运行脚本并提交新的 `data/latest.json`。

默认情况下，脚本使用公开新闻证据和关键词规则更新。如果你在 GitHub 仓库的 `Settings → Secrets and variables → Actions` 里添加 `OPENAI_API_KEY`，脚本会额外调用 AI 生成中文摘要，并分别判断定价、库存、产能信号。

股市信息由 `scripts/update-stocks.mjs` 生成 `data/stocks.json`。默认使用 Yahoo Finance chart 的公开非官方接口，适合原型和延迟行情观察；它可以返回日线 OHLC 数据，因此可以画走势图或 K 线图。如果你在 GitHub Secrets 里添加 `FINNHUB_API_KEY`，脚本会优先使用 Finnhub 获取美股报价，同时仍保留 Yahoo 日线历史作为走势图来源。生产级、实时、稳定、合规、多市场覆盖的行情通常需要付费 API。

当前计划任务是 UTC 21:20 运行，也就是日本时间每天 06:20 左右。

手动触发方式：

1. 打开 GitHub 仓库页面。
2. 进入 `Actions`。
3. 选择 `Update semiconductor tracker data`。
4. 点击 `Run workflow`。
5. 等待任务完成后，GitHub Pages 会读取新的 `data/latest.json`。

## 后续需要准备的数据

如果要把它从“网页原型”升级成真正的追踪工具，建议先准备一张表，每行一个产品节点：

```text
日期, 产业环节, 产品/工序, 公司, 地区, 定价数值, 定价单位, 定价变化, 库存数值, 库存单位, 库存变化, 产能数值, 产能单位, 产能变化, 来源链接, 可信度, 备注
```

后续可以把这张表做成 CSV/Excel，再让网页自动读取；也可以接入 RSS、公司公告、行业研究机构、新闻 API 或付费价格数据库。

## 重要说明

当前内置内容是追踪框架和区间估算示例，不是实时新闻、实时行情或投资建议。页面里写了整理日期，避免把静态信息误认为最新数据。后续接入真实数据时，应给每个数值补上来源、日期和可信度。
